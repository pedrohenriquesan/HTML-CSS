Cores

Representação por nome

A representação por nome é simples, é literalmente só inserir o nome da cor, o próprio VScode apresenta muitas opções de apoio, no entanto, esse modo não é indicado pois é muito limitado, uma vez que, existem milhões de cores.

Para entendermos as outras três representações devemos entender como as cores são formadas, um termo muito famoso é importante neste tópico, rgb significa red, green e blue, as três cores primárias que formam todas as outras a medida que são adicionadas ou retiradas, em suma, cada uma das três variam de 0 a 255 e, a cada valor que cada uma assume forma-se uma nova cor. Com isso, podemos ver as outras representações.

Representação por código hexadecimal

O código hexadecimal como o nome já diz é na base numérica hexadecimal e é composto por (#nnnnnn, n são dígitos e o # é fixo), é simples, cada cor primária do rgb possui dois números no código, onde 0 é nada e f é tudo. No exemplo que será mostrado a frente usamos o azul, onde temos a ausência total de vermelho e verde, enquanto temos a presença total do azul, com isso o código é #0000ff.

Representação por código rgb

Partindo agora para o terceiro modo de representação, o rgb, esse modo basicamente indica a quantidade de cada cor primária de 0 a 255, do seguinte modo: rgb(vermelho, green, blue), no caso do azul temos rgb(0, 0, 255).

Representação por hsl

O quarto e último modo de representação é o hsl (hue, saturation e luminosity), o mesmo não é muito utilizado, basicamente funciona de forma análoga ao rgb, mas aqui coloca-se os valores do matiz, da saturação e da luminosidade da cor.
